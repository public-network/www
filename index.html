Introduction

Who you are. What you own. What you say. What you trade. What you authorize. These five categories of fact underpin any functioning society, and in the digital world all five have been captured by a small number of private companies. They are recorded on corporate servers that can change the rules, restrict access, or remove records at any time. There is no public ledger for these facts. There is no common infrastructure that any application can read from and write to without permission.

Public Network is a protocol that records these five facts onchain: identity, asset, information, transaction, and authentication. It is an open set of schemas that anyone can read, that a curated set of writers can attest to, and that no participant can modify, remove, or gate. Applications are built on top; the protocol itself holds no position on what they do.

The minimalism is deliberate. A public good must be small enough to audit, simple enough to verify, and narrow enough to stay neutral. Applications read and write the five primitives through an open-source SDK. The protocol itself is ownerless, permanent, and built onchain.

Identity

Who they are.

A record of an entity that holds a cryptographic key: an individual, a group, an institution, or an autonomous agent. Today, identity is rented from platforms. A social account, an email login, a government handle. Each can be revoked, suspended, or deleted. On Public Network, an identity is bound to the keys that created it and works across every application on the protocol. It cannot be taken away.

Identities are ERC-721 tokens (PNI). Each identity owns a primary handle, a set of linked wallets, and an avatar. External accounts (Google, X, GitHub, email) may be linked to extend reputation across systems.

~/my-app · identity 
// Claim an identity handle
import { claimIdentity } from '@publicnetwork/sdk/identity';

const identity = await claimIdentity({
  handle: 'alice',
  wallet: signer,
});

// → PublicNetworkIdentity #1024 · 0xAlice…

Asset

What they own.

A record of ownership over a resource: property, art, data, intellectual work. Value in these resources depends on provable origin. Where did it come from, who made it, what is it composed of. Most digital assets today carry no such record. Generative systems produce artifacts in seconds with no origin trail. On Public Network, every asset is attested at creation and every transfer is recorded. Composition is tracked. Royalties route automatically.

Assets are ERC-721 tokens (PNA) with optional royalty routing via ERC-2981. The 1% protocol fee on secondary sales routes to the Public Network treasury automatically.

~/my-app · asset 
// Tokenize an asset with provenance
import { tokenizeAsset } from '@publicnetwork/sdk/asset';

const asset = await tokenizeAsset({
  identity,
  mediaUri: 'ar://bafy…',
  composedOf: [parentAssetId],
});

// → PublicNetworkAsset #4271 · composed

Information

What they say.

A record of content signed by an identity: a post, an article, a claim, a piece of research. Where it is published today determines whether it survives, whether it can be found, and whether it can be attributed. When the platform disappears or changes policy, the record is lost. On Public Network, information carries a cryptographic signature, a timestamp, and an immutable reference to its author.

Information lives inside Spaces (PNS), addressable containers for threads, publications, or communities. Anyone can create a Space.

~/my-app · information 
// Create a signed post
import { post } from '@publicnetwork/sdk/information';

const signed = await post.sign({
  identity,
  body: 'Hello, Public Network.',
  space: 'announcements',
});

const pending = await post.createDirect(signed);
await pending.awaitConfirmation();

Transaction

What they did.

A record of value exchange between identities: a sale, a listing, a coin launch, a cross-chain settlement. The digital economy today runs through intermediaries that take twenty to thirty percent of every sale and can deny service at any time. On Public Network, transactions are recorded at the protocol layer. Each transaction carries the signatures of its participants; fees route at the protocol layer.

The protocol fee is fixed at 1% of transaction value. Hard-coded, non-configurable. It flows to the Public Network treasury and funds ongoing development, audits, and infrastructure. Applications may layer additional fees; the creator retains the remainder.

~/my-app · transaction 
// Launch a coin with a 3-way fee split
import { launchCoin } from '@publicnetwork/sdk/transaction';

const coin = await launchCoin({
  identity,
  name: 'Example',
  symbol: 'EXMPL',
  totalFeeBps: 250,    // 2.50% total
  appCutBps: 50,        // 0.50% to the app
  // protocol takes 1.00% (fixed), creator keeps 1.00%
});

Authentication

What they authorized.

A record of an authorized action: a sign-in, a delegation to an agent, an approval of a third-party application. Most authentication today runs through servers owned by a handful of companies that control which applications can access which users. On Public Network, an identity signs a scoped authorization directly. Each authentication is a cryptographic signature from an identity attached to a scoped permission.

An identity may delegate scoped, time-bound authority to another key and revoke it at any time. Every delegation and revocation is recorded; the chain of authority is always walkable.

~/my-app · authentication 
// Delegate limited authority to an agent
import { delegation } from '@publicnetwork/sdk/authentication';

const d = await delegation.create({
  from: identity,
  to: agentKey,
  scope: ['post.create'],
  expires: '2026-12-31',
});

// Revoke any time
await delegation.revoke(d.uid);

Architecture

Public Network is built on Ethereum and settles on Optimism, an Ethereum L2 that provides fast, low-cost, permanent records.

Public Network does not run its own chain, does not operate its own consensus, and does not issue its own currency. It inherits the security and neutrality of Ethereum.

Schemas

The five primitives are realized as 20 open source schemas on EAS, registered once on Optimism. Each schema is public, permissionless to read, and writable by the per-schema allowlist. Writers are curated at launch; reads are universal.

Schemas compose. An asset can carry information. A space contains information. A transaction attests value exchange between identities. An authentication records the authority behind each.

Schemas are immutable once registered. No participant can modify, remove, or gate them.

Build on Public Network

Applications read and write the five primitives directly via the SDK. No permission, registration, or endorsement is required.

zsh 
$ pnpm add @publicnetwork/sdk

$ cat > hello-pn.ts <<'EOF'
import { configure } from '@publicnetwork/sdk';
import { claimIdentity } from '@publicnetwork/sdk/identity';
import { post } from '@publicnetwork/sdk/information';

configure({ network: 'optimism' });

const id = await claimIdentity({ handle: 'alice', wallet });
await post.createDirect(
  await post.sign({ identity: id, body: 'hello' })
);
EOF

$ pnpm tsx hello-pn.ts

Five modules mirror the five primitives: identity, asset, information, transaction, authentication. Core plumbing including EIP-712 signing, relay failover, and pending-attestation confirmation sits underneath.

Sign in with Public Network

A decentralized single sign-on primitive. An identity that works across every application built on Public Network. Wallet-first, portable, revocable.

The user’s wallet signs a scoped authorization payload. The application receives a signed attestation naming the identity, the requested scopes, and the expiration. No central authentication server is required.

~/my-app · sign-in 
// Request sign-in from a user
import { authentication } from '@publicnetwork/sdk/authentication';

const session = await authentication.signIn({
  app: 'my-app.example',
  scopes: ['identity.read', 'post.create'],
});

// session.identity · session.expires · session.signature

Patrons

Decentralize and open source Humanity.

Public Network is a permanent neutral ground in which to reside onchain. Patrons protect, support, and evolve it, one initiative at a time.

Any individual, group, or institution can become a Patron by contributing to Public Network: writing code, designing, translating, moderating, documenting, running infrastructure, or starting an initiative.

Autonomous agents acting on behalf of Patrons contribute alongside them.

The Perfection of the Imperfect Circle

Feeling insecure and alienated from others, the Imperfect Circle tried to join the ranks of the perfect circles, but over time its true self started to diminish along with the traces of its imperfect shape.

After the long quest for an ideally manufactured line, the Imperfect Circle realized that perfection was an abstraction and utopia, an elusive ideal with many interpretations that was impossible to achieve. Eventually, the purest form of perfection was found in its stead: authenticity with all its flaws and mistakes. The Imperfect Circle came back to its innermost being, its original form, unique, real, and free, accepting the innate beauty of itself and others.

But its imperfect shape can only be seen and appreciated by those who are able to look deeper and recognize its true form.
